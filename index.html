Bienvenido

Las ramas en Git y GitHub permiten trabajar en paralelo sobre un proyecto sin afectar el código principal. La rama principal (main/master) almacena el código estable, mientras que las ramas secundarias se crean para desarrollar nuevas características o corregir errores. Los cambios en las ramas secundarias se integran a la principal mediante una fusión (merge). En GitHub, las fusiones suelen hacerse a través de pull requests, que permiten revisar el código antes de unirlo al proyecto principal. Esto facilita la colaboración y el control de versiones. Además, el comando git log es útil para visualizar el historial de confirmaciones del repositorio. Este comando muestra información detallada sobre cada commit, como el hash, el autor, la fecha y el mensaje asociado. Puedes usar parámetros como --oneline para ver un resumen breve, --graph para visualizar un gráfico de las ramas, o --since y --until para filtrar por rango de fechas.
