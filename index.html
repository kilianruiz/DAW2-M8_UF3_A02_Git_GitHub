Bienvenido

Las ramas en Git y GitHub permiten trabajar en paralelo sobre un proyecto sin afectar el código principal. La rama principal (main/master) almacena el código estable, mientras que las ramas secundarias se crean para desarrollar nuevas características o corregir errores. Los cambios en las ramas secundarias se integran a la principal mediante una fusión (merge). En GitHub, las fusiones suelen hacerse a través de pull requests, que permiten revisar el código antes de unirlo al proyecto principal. Esto facilita la colaboración y el control de versiones.
